This explorer needs WebGL, and your browser couldn't create a 3D context. This usually means graphics acceleration is unavailable (common in virtual machines) and the browser refuses to fall back to software rendering.

Ways in, fastest first:

1. Relaunch Chrome as: google-chrome --enable-unsafe-swiftshader (software rendering — works anywhere, modest frame rate)
2. Try Firefox, which still allows software WebGL by default
3. In a VM: enable 3D acceleration (VirtualBox: Settings → Display → Enable 3D Acceleration + 128 MB video memory, with Guest Additions installed), then restart the VM
